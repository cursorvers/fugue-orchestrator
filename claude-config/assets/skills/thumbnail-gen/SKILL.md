# /thumbnail - サムネイル・バナー生成スキル v2

KAWAI式「構造化プロンプト3階層」+ 「制作前7つの掟」+ **スタイルライブラリ10分類** + **コンセプトファースト設計** + **品質ゲート3段階** に基づくサムネイル生成。

Trigger: /thumbnail, サムネイル, バナー, アイキャッチ, OGP, thumbnail, banner

---

## アーキテクチャ v2

```
[入力] → [Phase 0: スタイル選択] → [Phase 1: 7掟] → [Phase 2: コンセプト設計]
                                                         → [Phase 3: プロンプト合成]
                                                         → [Phase 4: 生成+品質ゲート]
出力: SVG(0円) / Gemini NB2(~1円) / Manus(高品質) / Freepik(20円)
```

## Phase 0: スタイル選択

ユーザー入力からスタイルカテゴリを自動判定 or 明示選択。

| ID | スタイル | 適用場面 | 代表例 |
|----|----------|----------|--------|
| **A** | シンプル宣言型 | 告知・実績報告 | 「800名突破！」「書き初め」 |
| **B** | エージェントWF型 | ターゲット分析→自動設計 | 2段階AI分析フロー |
| **C** | 記事ベース型 | 記事内容→自動抽出 | note記事ヘッダー自動生成 |
| **D** | リクエストベース型 | 依頼文→構造化 | 「冬の保湿液の記事」→プロンプト |
| **E** | エディトリアル/雑誌型 | 権威・信頼・プレミアム | NewsPicks/WIRED/Forbes風 |
| **F** | タイポグラフィック型 | 文字のみ・ミニマル | 「絶滅」「99%消した」 |
| **G** | プロダクト/テック型 | ツール・機能紹介 | ターミナルUI/フラットアイコン |
| **H** | データビジュアル型 | 統計・数字インパクト | 「$600B負けている」「14件更新」 |
| **I** | コンセプトメタファー型 | 抽象概念の視覚化 | ロウソクvs LED/ガラス球 |
| **J** | コラージュ/ミックス型 | 複合表現・ZINEスタイル | 雑誌BRUTUS風/手描き混在 |

### スタイル自動判定ルール
- 数字/実績を含む → **A** or **H**
- 記事URL/本文を含む → **C**
- 抽象テーマ/哲学的 → **F** or **I**
- ツール名/技術名を含む → **G**
- 「〜風」「〜のような」を含む → **E** or **J**
- ターゲット情報あり → **B**
- それ以外 → **D** (汎用)

## Phase 1: 制作前フロー（7つの掟）

### Phase 1-B: タイトルパターン連動スタイル選択 (KAWAI式 + Hook統合)
記事タイトルが提供されている場合:
1. **タイトルパターン判定** (T1-T8: 常識否定/あるある共感/実用公開/速報/数字/権威/問題提起/BA)
2. **フック型推論**: `titleHookMapping` からタイトルパターンに最適なフック型(H1-H6)を取得
3. **視覚優先度決定**: `hookVisualMapping` でフック型→レイアウト/タイポグラフィ優先度を特定
4. **Pillar検出**: `pillarDetect` でP1/P2/P3を判定し、スタイル候補を絞込み
5. `titleStyleMapping` + Pillar候補 + フック型 styleAffinity の交差から最適スタイルを選択
6. タイトルパターンが不明な場合は従来のSTYLE_RULES判定にフォールバック

**フック型→視覚表現の対応** (prompt-library.json `hookVisualMapping` 参照):
| Hook | 視覚優先度 | レイアウト推奨 |
|------|-----------|--------------|
| H1 (PQR) | 問題描写を最大化 | L1, L5 |
| H2 (Open Loop) | 結果/数字を巨大表示 | L3, L4 |
| H3 (逆説) | 視覚的逆転・認知的不協和 | L4, L8, L10 |
| H4 (In Medias Res) | シネマティック場面描写 | L7, L2 |
| H5 (If-Then) | ターゲット直接指名 | L1, L5 |
| H6 (数字リード) | 数字が画面支配(25-35%) | L4, L3, L9 |

サムネイル制作の前に、以下を**必ず**言語化する。
ユーザーが明示していない項目は推論で補完し、確認する。

| # | 掟 | 問い | 例 |
|---|-----|------|----|
| 1 | 目的定義 | このサムネの目的は？（行動レベル） | 「noteでクリックさせる」 |
| 2 | 状況ターゲット | 誰がどんな状況で見る？ | 「通勤電車でスマホスクロール中」 |
| 3 | 感情トリガー | 一瞬で何を感じさせる？（1つだけ） | 「当事者意識（自分のことだ！）」 |
| 4 | 1メッセージ | 伝える情報は？（1つだけ） | 「30%はAIが書いた」 |
| 5 | 文字は認識 | テキストは読むのではなく0.2秒で認識させる | 7-13文字、短く刺す |
| 6 | 表示コンテキスト | どこで・どのサイズで見られる？ | 「note 16:9、160x90px縮小表示」 |
| 7 | ここで初めてデザイン | 上記が決まってからツールを開く | — |

## Phase 2: コンセプト設計（B/E/I は必須、他はオプション）

**Phase 2 必須スタイル**: B(ペルソナ分析), E(エディトリアル — コンセプト深度必要), I(メタファー — 抽象概念の言語化必要)
**Phase 2 推奨スタイル**: C(記事が長文の場合), F(逆説/哲学テーマの場合)

### 2a. ユーザーニーズ9段階分類

ターゲットの購買心理段階を特定し、アプローチを決定する。

| Lv | 状態 | アプローチ |
|----|------|-----------|
| 1 | 無関心・未認知 | 違和感フックで注意を引く |
| 2 | いずれ不要と思っている | 危機感を煽る |
| 3 | 必要だが行動しない | 行動の簡単さを示す |
| 4-5 | 情報収集・比較中 | 具体的メリット・数字で差別化 |
| 6-7 | 購入済・満足 | 次のステップを提示 |
| 8-9 | 不満・乗り換え検討 | 優位性を論理的に示す |

### 2b. 3つの認知フック

| フック | 手法 | 視覚表現 |
|--------|------|----------|
| **違和感・好奇心** | 常識外れなビジュアルで脳を刺激 | 意外な組み合わせ/スケール逆転 |
| **興味・感情共感** | 「自分のことだ」と思わせる | ターゲットの悩みを描写 |
| **論理的価値** | 数字・権威性で行動を促す | 大きな数字/ロゴ/肩書き |

### 2c. コピー設計（自動生成時）

1. メインコピー候補を3案生成（7-13文字、2行構成）
2. サブコピー候補を各案に付与（20文字以内）
3. カテゴリの感情トリガーと照合して最適案を選択
4. **選択理由を明記してから**次フェーズへ進む

## Phase 3: 構造化プロンプト合成（3階層 + レイアウト + 品質仕様）

### 階層1: 掲載する内容 (The "What")
画像内に**文字として配置**される要素のみ。

```
# 画像に掲載するテキスト:
## メインコピー:
- 1行目: "{7-13文字}"
- 2行目: "{7-13文字}"
## サブコピー:
- "{20文字以内}"
## 補足（任意）:
- "{日時/数値/CTA/バッジ}"
```

### 階層2: アイキャッチ & 背景 (The "Subject")

```
# アイキャッチ:
- {被写体の具体的描写}
- {スタイル: フラットイラスト/写真/3D/ピクセルアート/水彩/劇画 等}
- {ポーズ・表情・服装の指定（人物の場合）}

# 背景:
- {背景の具体的描写}
- {密度・テクスチャの指定}
```

**人物指定ルール:**
- Manus使用時: 上半身のみ、5-7頭身、リアル描写、【人物描画の絶対ルール】セクション必須
- Gemini使用時: イラスト風推奨、簡体字防止句必須
- HTML/SVG: 人物イラスト禁止 → アイコン/図形/テキストのみ

### 階層3: デザイン (The "How")

```
# フォント:
- メインコピー: {具体的フォント名 + ウェイト + スタイル}
- サブコピー: {具体的フォント名}
  ※ Noto Sans JP Black/Bold を基本。Regular禁止

# アスペクト比:
- {16:9(1280x670) / 1:1 / 4:5 / 9:16 / 3:2 / 3:4}

# 配色:
- 背景（70%）: {HEX}
- メインカラー（25%）: {HEX}
- アクセントカラー（5%）: {HEX}
- ホワイトバランス: {色温度K（任意）}
  ※ 3色+グレー階調が上限。5色以上禁止

# レイアウト: → レイアウトパターン参照
- {選択パターン名}
- {具体的配置指示}

# スタイル:
- {印象キーワード 3つまで}

# 仕様:
- 画質: 4K高画質
- 解像度: {具体的ピクセル数}

# 禁止事項:
- {エンジン共通 + スタイル固有の禁止事項}
```

| # | パターン | 構成 | プロンプト要素 | 適合スタイル |
|---|----------|------|---------------|-------------|

| L4 | **日の丸構図** | 中央にキービジュアル | centered composition, symmetrical, bullseye | F,G,I |
| L5 | **Z字パターン** | 左上→右下視線誘導 | Z-pattern visual flow, headline top-left, CTA bottom-right | D,H |

| L7 | **オーバーラップ** | 文字と画像が重なる | overlapping elements, 3D depth, text behind person | E,J |
| L8 | **余白重視** | 大きな空白+要素集中 | minimalist, generous negative space, clean | F,G,I |
| L9 | **グリッド** | 格子状に複数要素 | grid system, modular layout, multiple frames | H,J |

| 技法 | 効果 | プロンプト指示 |

## タイポグラフィシステム

### 基本ルール
- **ジャンプ率**: メインコピー ≧ サブコピーの1.5倍（推奨3倍以上）

- **余白**: キャンバスの30%以上（F型は35%以上）
- **文字間**: やや広め（0.3px程度）

## Phase 4: 生成 + 品質ゲート

### 生成エンジン優先順位（v2.4 自動ルーティング）

```
人物あり: Manus API (v4構造, 90s timeout) → Gemini NB2 → Gemini Legacy
人物なし: Gemini NB2 → Gemini Legacy
フォールバック: SVG → HTML Template → Freepik
```

**自動判定** (`shouldUseManus()`): プロンプト内人物キーワード or カテゴリpersonフィールド or スタイルA/B/C/E/I → Manus優先
**Manus不在時**: `MANUS_API_KEY` 未設定 → Geminiのみで動作（人物品質は低下）

### Gemini NB2 呼び出し（メイン経路）

```bash
# note-thumbnail-gen.js経由
node claude-config/assets/skills/note-generate/scripts/note-thumbnail-gen.js \
  --prompt "$(cat prompt.txt)" \
  --output ./thumbnail.png
# → 1280x670 PNG, 最大6回リトライ (2モデル×2リトライ+fallback)
```

### Manus v4 人物描画ルール（人物あり時は必須セクション）

```
【人物描画の絶対ルール】
- 人物は上半身のみ（バストアップ～ウエストアップ）
- 5〜7頭身のリアルな等身（デフォルメ禁止）
- 顔パーツ（目・鼻・口・眉）を明確に描画
- 手足の歪み禁止、アクセサリーはテキスト領域にはみ出さない
- 棒人間・アイコン風・絵文字風は絶対不可
- 背景はベタ塗りのみ（リップル/放射/グロー効果禁止）
- 高解像度: 1200x675px以上, PNG, 200KB以上
- 低解像度や小さいファイルは不可
```

### 品質ゲート3段階

**Gate 1: 自動チェック（即時）**
- ファイルサイズ ≧ 100KB（< 100KB → 即再生成）
- 推奨: ≧ 200KB
- 解像度: 指定サイズ以上
- フォーマット: PNG

**Gate 2: 視覚検査（自動化 — `thumbnail-gate2.js`）**
フルサイズ画像(1280x670)をGemini Flashに送信し、160x90px表示時の可読性を自動判定。
- `mainTextReadable`: メインコピーが160x90pxで判読可能か
- `contrastSufficient`: テキスト-背景コントラストが十分か
- `simplifiedChineseDetected`: 簡体字が混入していないか
- fail-open: API失敗時はスキップ (パイプラインをブロックしない)
- `mainTextReadable=false` の場合のみ1回リトライ

**Gate 3: 再生成プロトコル**
- 同一プロンプト再試行: 最大1回
- 2回失敗: プロンプトを書き換え
- 3回失敗: 生成エンジンを変更（Manus → Gemini → SVG）
- 品質不合格の具体的理由を記録

## スタイル別テンプレート

### Style A: シンプル宣言型
```
# テキスト:
- サブテキスト: {文脈・カテゴリ}
- メインテキスト: {数字+感嘆符 or 短いメッセージ}
- サブテキスト2: {補足メッセージ}

# アイキャッチ:
- {数に合わせた人物群 or シンプルなモチーフ}
- 人物はフレームからはみ出すほど大きく配置（はみ出し技法）

# レイアウト: [テキスト: 上部, アイキャッチ: 中央〜下部]
# 配色: ビビッドソリッド背景 + 白文字
```

## 160x90px サムネイル視認性チェック

全てのサムネイルは160x90pxに縮小された状態で以下を満たすこと:

- [ ] メインコピーの最重要ワード（2-4文字）が判読可能
- [ ] 背景色とテキスト色のコントラストが十分
- [ ] 2つのフォーカルポイント（テキスト + ビジュアル要素）が識別可能

- [ ] 人物（カテゴリ対応アーキタイプ、はみ出し表現あり）
- [ ] コピー（7-13文字、1文字アクセント色変え）

## 実行手順

1. ユーザーの入力を受け取る
2. **Phase 0**: スタイル自動判定 or 選択
3. **Phase 1**: 7つの掟を埋める（推論 + 確認）
4. **Phase 2**: コンセプト設計（Style B時は必須、他は簡略可）
