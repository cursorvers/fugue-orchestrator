# Gemini CLI Critique Prompt

Review this mobile-first web app information architecture.

Focus on:

- iPhone usability
- Pixel usability
- one-thumb navigation
- tab naming clarity
- whether `Happy` as the conversational home is intuitive
- whether `Now`, `Tasks`, `Alerts`, and `Recover` are visually distinct
- whether progress and outputs are legible without opening GitHub
- whether recovery controls are safe and not overpowered

Challenge:

- information density
- terminology
- touch target sizing
- empty-state clarity
- whether the app feels like a single front rather than stitched infrastructure
