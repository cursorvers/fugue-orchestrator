# Gemini CLI Critique Prompt

Critique this mobile-first `Kernel` operator app.

Focus on:

- iPhone usability
- Pixel usability
- one-thumb navigation
- tab naming clarity
- progress readability
- output readability
- whether `Recover` feels safe and bounded
- whether `Happy` feels like a conversational home instead of a hidden debug tool
- whether the app feels like one coherent product rather than stitched infrastructure

Challenge:

- hierarchy
- spacing
- touch targets
- information density
- empty states
- how clearly `Happy.app` sits inside the outer app
