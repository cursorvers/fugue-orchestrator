# Pencil Brief

Design the outer all-in-one mobile app for `Kernel`.

This app is:

- mobile-first
- iPhone-safe before Android-optimized
- one-handed by default
- visually calm but operationally alive
- the outer shell that contains `Happy.app` as the conversational surface

Screens:

1. `Happy`
2. `Now`
3. `Tasks`
4. `Alerts`
5. `Recover`

Must show:

- current route: `local`, `GHA continuity`, or `FUGUE rollback`
- current task and phase
- task outputs
- bounded recovery controls
- degraded/fallback state
- a visible sense that `Kernel` is alive without looking like a raw admin dashboard

Avoid:

- desktop dashboard density
- provider-centric UI
- fake precision progress
- noisy alert spam
- purple-on-white defaults
